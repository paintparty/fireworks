# ADR 0006: Golden test suites live in hifi and call (hifi x opts)

Date: 2026-07-09 · Status: accepted

## Context

The fireworks golden suites were generated by `write-tests-ns!` and exercised the engine through the `?` macro (`(-> (? :data opts x) :formatted :string)`). Ported as-is, hifi's tests would depend on fireworks — a circular repo relationship.

## Decision

The generator and golden suites move to hifi and emit `(-> (hifi qv merged-opts) escape-sgr string/join)`. Macro-specific tests (`basic-tapping-macros-tests`, file-info/label annotation tests) stay in fireworks, hand-written in `fireworks.core-test`.

Golden values are generated on the JVM and asserted on JVM + babashka. They are NOT run in cljs — regex/fn/transient representations differ per platform (verified: 8/38 fail in cljs for representation reasons only). cljs coverage is compile + `hifi.core-test` under karma. Follow-up: generate cljs-specific goldens.

## Consequences

- hifi self-tests with no fireworks dependency.
- fireworks' test surface shrank to macro behavior: tapping passthrough, annotation templates, print-with/log? passthrough (JVM + bb + cljs karma).
